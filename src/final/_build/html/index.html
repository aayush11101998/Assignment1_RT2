assignment3 Module¶

The node provides user with the possibility to select the robot’s behaviour in the Gazebo simulation environment

param /robot_state

param used in the state() function

scripts.assignment3.main()[source]¶

The main allows to set the parameter robot_state according to the users choice.

scripts.assignment3.state()[source]¶

Function allows user to give a char input to robot so that it choses a mode to run in the simulation.

There are three behaviours from which a user can select:

1 - Teleop keyboard

2 - assistance to avoid collisions, advertised by teleop_keyboard

3 - movebase client, advertised by move_base

move_base Module¶

Service:

/move_base (action client)

The node provides user with the option to move the robot autonomously to a selected point defined by user without collision in the gazebo simulation environment.

scripts.move_base.main()[source]¶

Here the move_base node is initialized, if the robot_state parameter is for move_base is activated.

scripts.move_base.movebase()[source]¶

Movebase(an action client) gives the user chance to define goal for the robot by asking them to define certain point coordinates which are set as Movebasegoal() coordinates. More about Action_Client. It also gives user some insights like the total distance that robot has to cover and robots current position coordinates.

teleop_keyboard Module¶

Subscribes to:

/scan /odom

The node provides user with the options to control the robot using teleop keyboard as well as assist user of a collision free teleop keyboard control in the Gazebo simulation environment.

class scripts.teleop_keyboard.PublishThread(rate)[source]¶

Publishes thread to update the pose of robot and allows user to have possibility to control the robot by keyboard keys and also can assist them to avoid obstacles. Once the task finishes the thread resets everything and stops the twist messages.

run()[source]¶

Method representing the thread’s activity.

You may override this method in a subclass. The standard run() method invokes the callable object passed to the object’s constructor as the target argument, if any, with sequential and keyword arguments taken from the args and kwargs arguments, respectively.
